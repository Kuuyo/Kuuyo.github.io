DigDug

A small 2D engine started from Minigin with DigDug as example game.
Minigin is an SDL-based mini engine with GLM.

The engine and game is entirely C++ code based, no editor. The Engine is first built as a lib and included in the game project in the build process.

The game uses Box2D as physics engine and has an InputManager that supports both keyboard/mouse and controller.

It also includes a ResourceManager that loads and stores textures and fonts. Additionally it can also load in JSON, and this was used to load in levels for DigDug.

The engine uses certain Game Programming Patterns such as Observer/Subject and Finite State Machine.

It also uses the component system and the GameObject these are attached to is Final.

The Game Loop features a FixedUpdate, Update, LateUpdate and Render. But the Render is only used internally, as any graphical components get stored seperately since this is only a 2D engine, to prevent every GameObject being iterated again in the render stage.
