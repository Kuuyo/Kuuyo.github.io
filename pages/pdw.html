PerforceDiscordWebhook

When developing Punch along with four others, we used Perforce as our source control and Discord as our communication platform.

Multiple times in the process I wanted to check things on Perforce on my phone while not at a laptop, for things such as being able to quickly look at some recent file changes and answer a question someone asked me while I was for example on a train.

This resulted in me developing a tool with the goal to get Perforce changes with changelists on Discord.

It can run both on Windows and Linux, as I deployed it to run on Heroku.

Once it detects changes, it constructs a JSON file and sends it to a Discord Webhook through curl.

Then additionally it generates a html file of the diff2 obtained from Perforce.

When we were close to the deadline of Punch, I got notified there was a new changelist. When I went to look what it was, I saw that a couple of files had been deleted.
I quickly launched the game and some UI elements were gone. I quickly reverted the deletion on Perforce and was happy I created and set up the bot.
